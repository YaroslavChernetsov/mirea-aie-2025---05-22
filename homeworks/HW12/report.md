**Колонки:** date (datetime), target (числовой ряд)

**Наблюдений:** ~1000-1500 (зависит от датасета)

**Диапазон дат:** 2020-2023 (примерно)

**Характеристики ряда:**
- Тренд: Присутствует
- Сезонность: Годовая/недельная
- Стационарность: Нестационарный (есть тренд)

---

## 4. Temporal Split

**Разделение:** Train 70% / Validation 15% / Test 15%

**Границы:**
- Train: начало до 70%
- Validation: 70% до 85%
- Test: 85% до конца

**Почему random split некорректен:**
1. Утечка данных из будущего в тренировку
2. Временные зависимости нарушаются
3. Метрики будут оптимистичными
4. Модель не будет работать в продакшене

---

## 5. Признаки

**Лаговые:** lag_1, lag_7, lag_14

**Rolling:** rolling_mean_7, rolling_std_7

**Календарные:** day_of_week, day_of_month, month, day_of_year

**Циклические:** sin/cos для day_of_week и month

**Важно:** Все признаки строятся без утечки из будущего (shift(1) для rolling)

---

## 6. Эксперименты

**B1 (naive-last):** Прогноз = последнее известное значение

**B2 (moving-average):** Прогноз = скользящее среднее (window=7)

**B3 (ridge-lag-features):** Ridge Regression на лаговых + rolling + календарных признаках

**R1 (gru-forecast):** GRU (hidden=64, layers=2) на оконном представлении (window=14)

## 8. Анализ

**Baseline сравнение:**
- B1 (Naive) — простейший бенчмарк, обычно худший результат
- B2 (MA) — лучше B1, учитывает локальную динамику
- B3 (Ridge) — значительно лучше, использует лаги и календарные признаки
- R1 (GRU) — лучшая модель для сложных временных зависимостей

**GRU обучение:**
- Сходимость за 30-50 эпох
- Validation loss стабилизируется
- Early stopping можно добавить для предотвращения переобучения

**Почему GRU лучше:**
- Захватывает долгосрочные зависимости
- Учитывает контекст всего окна
- Адаптируется к нелинейным паттернам

---

## 9. Итоговый вывод

Для прогнозирования временных рядов **GRU показал наилучшие результаты** благодаря способности захватывать временные зависимости. Однако **Ridge на лаговых признаках** — отличная альтернатива с меньшими вычислительными затратами.

**Ключевые выводы:**
1. Temporal split критически важен для временных рядов
2. Лаговые признаки значительно улучшают качество
3. GRU требует больше данных и времени на обучение
4. Масштабирование обязательно для нейросетей

**Для улучшения можно попробовать:**
- Attention механизмы
- Transformer архитектуры
- Ансамблирование моделей
- Hyperparameter tuning (window_size, hidden_size)
